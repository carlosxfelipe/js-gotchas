# JavaScript: As Pegadinhas que Todo Dev Cai

## Introdução

O JavaScript é a linguagem mais onipresente do mundo, mas sua imensa flexibilidade muitas vezes se torna um verdadeiro campo minado.

Como a linguagem prefere tentar "adivinhar" as intenções do desenvolvedor ao invés de interromper a execução com erros rigorosos, é comum nos depararmos com bugs silenciosos e comportamentos exóticos.

O que muitos consideram defeitos são, na verdade, mecânicas arquiteturais profundas. Portanto, este compilado não foca em fazer você decorar regras, mas sim em explicar os bastidores lógicos de cada uma dessas famosas "pegadinhas".

Seja bem-vindo, e boa sorte no Debug do ECMAScript!

> **Dica Visual:** Este guia utiliza a fonte **JetBrains Mono**. Ela possui suporte nativo a *ligatures* (ligaduras tipográficas), o que significa que alguns operadores (como `===`, `!==` ou `=>`) podem aparecer como símbolos únicos estilizados. Isso ajuda na clareza e na velocidade de leitura do código!
